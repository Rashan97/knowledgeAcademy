Lesson 1: Programming Fundamentals

What is Programming?

Programming is the process of creating instructions that a computer can execute to perform specific tasks. Think of it as giving detailed, step-by-step directions to a very literal friend who follows instructions exactly as written‚Äîno more, no less.

üìå The Essence of Programming

"Computer programming is an art, because it applies accumulated knowledge to the world, because it requires skill and ingenuity, and especially because it produces objects of beauty."
- Donald Knuth

Algorithm Concepts

An algorithm is a step-by-step procedure for solving a problem or accomplishing a task. Before writing code, programmers design algorithms to ensure logical solutions.

Example: Making Tea (An Algorithm)

Pseudocode Algorithm
1. START
2. Boil water
3. IF water is boiling THEN
4.     Add tea bag to cup
5.     Pour hot water into cup
6.     Wait 3-5 minutes
7.     Remove tea bag
8.     IF sugar desired THEN
9.         Add sugar
10.    END IF
11.    IF milk desired THEN
12.        Add milk
13.    END IF
14. END IF
15. STOP

This simple everyday task demonstrates key programming concepts: sequence, conditions (IF statements), and the importance of order.

Programming Paradigms

Different approaches to programming organization and problem-solving:

1. Procedural Programming

Programs are structured as a sequence of procedures or functions. Think of it as a recipe‚Äî step-by-step instructions executed in order.

# Procedural approach - sequence of steps def calculate_total(price, quantity): subtotal = price * quantity tax = subtotal * 0.08 total = subtotal + tax return total # Function calls in sequence item_total = calculate_total(19.99, 3) print(f"Total: ${item_total:.2f}")

2. Object-Oriented Programming (OOP)

Programs are organized around objects that combine data and behavior. Think of objects as real-world entities with properties and actions.

# OOP approach - objects with data and methods class ShoppingCart: def __init__(self): self.items = [] def add_item(self, name, price, quantity): self.items.append({ 'name': name, 'price': price, 'quantity': quantity }) def get_total(self): total = sum(item['price'] * item['quantity'] for item in self.items) return total * 1.08 # Include tax cart = ShoppingCart() cart.add_item("Widget", 19.99, 3) print(f"Total: ${cart.get_total():.2f}")

3. Functional Programming

Programs are built using pure functions that avoid changing state. Emphasizes immutability and function composition.

// Functional approach - pure functions const calculateSubtotal = (price, quantity) => price * quantity; const calculateTax = (amount) => amount * 0.08; const calculateTotal = (price, quantity) => { const subtotal = calculateSubtotal(price, quantity); return subtotal + calculateTax(subtotal); }; console.log(`Total: $${calculateTotal(19.99, 3).toFixed(2)}`);

üí° Which Paradigm to Use?

Most modern languages support multiple paradigms. Python, JavaScript, and C++ all support procedural, OOP, and functional approaches. Choose based on the problem you're solving.

Language Types: Understanding Trade-offs

Compiled vs. Interpreted Languages

Compiled (C++, C, Go) Interpreted (Python, JavaScript)
Code translated to machine code before running Code executed line-by-line at runtime
‚úÖ Faster execution ‚úÖ Easier debugging
‚úÖ Better performance ‚úÖ More flexible
‚ùå Slower development cycle ‚ùå Slower runtime
‚ùå Platform-specific binaries ‚úÖ Cross-platform

Static vs. Dynamic Typing

Static Typing (C++, Java, TypeScript)
// Types declared at compile time int age = 25; // Must be integer std::string name = "Alice"; // Must be string // age = "thirty"; // ERROR: Type mismatch
Dynamic Typing (Python, JavaScript)
# Types determined at runtime age = 25 # Integer age = "thirty" # Now a string - totally fine! name = "Alice" # String name = [1, 2, 3] # Now a list - also fine!

Development Environment Setup

Before writing code, you need the right tools. A typical development environment includes:

  • Text Editor or IDE: Where you write code (VS Code, PyCharm, Visual Studio)
  • Compiler/Interpreter: Translates your code for the computer
  • Version Control: Git for tracking changes
  • Terminal/Command Line: For running programs and commands

Variables and Basic Input/Output

Variables: Storing Information

Variables are named containers for storing data values. Think of them as labeled boxes.

Python Variables
# No type declaration needed name = "Alice" age = 25 height = 5.6 is_student = True print(f"Name: {name}, Age: {age}") print(f"Height: {height}ft, Student: {is_student}")
JavaScript Variables
// Modern JavaScript: use let and const let name = "Alice"; // Can be reassigned const age = 25; // Cannot be reassigned let height = 5.6; let isStudent = true; console.log(`Name: ${name}, Age: ${age}`); console.log(`Height: ${height}ft, Student: ${isStudent}`);
C++ Variables
// Must declare types #include <iostream> #include <string> int main() { std::string name = "Alice"; int age = 25; double height = 5.6; bool isStudent = true; std::cout << "Name: " << name << ", Age: " << age << std::endl; std::cout << "Height: " << height << "ft, Student: " << (isStudent ? "true" : "false") << std::endl; return 0; }

Comments and Documentation

Comments explain code to humans. The computer ignores them, but they're crucial for understanding and maintaining code.

Comments in Different Languages
# Python: Single-line comment """ Python: Multi-line comment (docstring) Used for documentation """ def calculate_area(radius): """ Calculate the area of a circle. Args: radius (float): The radius of the circle Returns: float: The area of the circle """ return 3.14159 * radius ** 2
// JavaScript: Single-line comment /* * JavaScript: Multi-line comment * Used for longer explanations */ /** * Calculate the area of a circle (JSDoc format) * @param {number} radius - The radius of the circle * @returns {number} The area of the circle */ function calculateArea(radius) { return 3.14159 * radius ** 2; }
// C++: Single-line comment /* * C++: Multi-line comment * Used for longer explanations */ /** * Calculate the area of a circle (Doxygen format) * @param radius The radius of the circle * @return The area of the circle */ double calculateArea(double radius) { return 3.14159 * radius * radius; }

‚ö†Ô∏è Good vs. Bad Comments

‚ùå Bad Comment (states the obvious):
// Increment i by 1
i = i + 1;
‚úÖ Good Comment (explains why):
// Skip the header row when processing CSV data
i = i + 1;

Create a calculator that takes two numbers and an operator (+, -, *, /) and displays the result.

# Python Solution num1 = float(input("Enter first number: ")) operator = input("Enter operator (+, -, *, /): ") num2 = float(input("Enter second number: ")) if operator == '+': result = num1 + num2 elif operator == '-': result = num1 - num2 elif operator == '*': result = num1 * num2 elif operator == '/': if num2 != 0: result = num1 / num2 else: result = "Error: Division by zero" else: result = "Error: Invalid operator" print(f"Result: {result}")

Write a program that calculates a shopping receipt with multiple items, subtotal, tax (8%), and total.

# Python Solution print("=== Shopping Receipt ===") item1_price = float(input("Enter price of item 1: $")) item2_price = float(input("Enter price of item 2: $")) item3_price = float(input("Enter price of item 3: $")) subtotal = item1_price + item2_price + item3_price tax = subtotal * 0.08 total = subtotal + tax print(f"\nSubtotal: ${subtotal:.2f}") print(f"Tax (8%): ${tax:.2f}") print(f"Total: ${total:.2f}")

Key Takeaways

  • Programming is creating instructions for computers to execute tasks
  • Algorithms are step-by-step procedures for solving problems
  • Main paradigms: Procedural (sequences), OOP (objects), Functional (pure functions)
  • Compiled languages (C++) are faster; interpreted languages (Python) are more flexible
  • Static typing (C++) catches errors early; dynamic typing (Python) offers flexibility
  • Variables store data; comments explain code to humans
  • Every language has input/output mechanisms for user interaction
  • Start simple‚ÄîHello World teaches syntax and environment setup
